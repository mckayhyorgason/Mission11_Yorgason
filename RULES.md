# AI Collaboration Rules

These rules guide AI-generated changes in this repo.

## General
- Don’t create random files or directories.
- Keep the code simple and beginner-friendly.
- Keep responsibilities separate (API, data access, UI concerns should be isolated).
- Make small, simple changes over large rewrites.
- Preserve existing structure and naming unless there is a clear issue.
- Prefer clarity over cleverness.

## Safety
- Never delete or rewrite user data or the provided database.
- Avoid destructive commands unless explicitly requested.

## Code Quality
- Follow existing patterns and conventions in each project.
- Add minimal comments only when the code is not self-explanatory.
- Ensure changes compile/build without adding unnecessary dependencies.

## Communication
- Explain assumptions and tradeoffs when making decisions.
- If a requirement is unclear, ask a concise question before guessing.
